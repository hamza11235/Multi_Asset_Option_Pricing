# Basket Option Pricing and Hedging Under Correlation Model Risk

This repository is a cleaned final submission on how correlation assumptions affect the pricing and hedging of a European basket call in a multi-asset Black-Scholes framework.

The project now has one coherent narrative:

1. build and test the synthetic pricing and hedging framework;
2. stress that framework to see when regime dependence matters;
3. extend the model to the more realistic case where the calm/stress regime is hidden and must be filtered from returns.

## Main Findings

The results support a careful version of the original thesis.

- Time-0 pricing differences between constant and regime-switching correlation are usually modest.
- Hedging is the main economic issue: both dynamic hedges reduce risk sharply relative to an unhedged short option.
- After putting the synthetic hedge comparisons on a common premium basis, the baseline oracle-versus-constant comparison is mixed rather than one-sided.
- Under harsher synthetic stress settings, regime-aware hedging becomes more relevant, especially in downside-tail metrics.
- In the latent-state follow-up, a filtered hedge does not match the oracle benchmark, but it still preserves a meaningful share of the oracle tail improvement when stress is more severe.

## Repository Layout

The tracked repo is organized into three top-level folders:

- [scripts](scripts): reusable Python helpers and notebook-generation utilities
- [notebooks](notebooks): the final notebook sequence
- [outputs](outputs): saved CSVs, artifacts, and final plots generated by the notebooks

## Notebook Sequence

### Synthetic Analysis

1. [01_setup_and_pricing.ipynb](notebooks/01_setup_and_pricing.ipynb)  
   Sets up the synthetic market, simulates constant and regime-switching paths, checks empirical correlations, and prices the basket option under both models.

2. [02_delta_hedging_backtest.ipynb](notebooks/02_delta_hedging_backtest.ipynb)  
   Runs the baseline delta-hedging experiment from scratch and compares unhedged, constant-correlation, and oracle regime hedges on a common premium basis.

3. [03_stress_sensitivity.ipynb](notebooks/03_stress_sensitivity.ipynb)  
   Focuses on the high-signal stress experiments: a clean benchmark matrix and a structural stress test with harsher calm/stress dependence.

4. [04_dimension_and_robustness.ipynb](notebooks/04_dimension_and_robustness.ipynb)  
   Studies homogeneous basket dimension scaling and a fixed-`N=10` robustness grid over the key stress parameters.

### Latent-Regime Follow-Up

5. [05_filtered_regime_hedging.ipynb](notebooks/05_filtered_regime_hedging.ipynb)  
   Keeps the true world regime-switching but hides the regime from the hedger. The hedger only sees past multivariate returns and filters a posterior stress probability, allowing a comparison between constant, oracle, and filtered hedges.

## Reproducing the Project

Use Python 3.10+ and install the dependencies from [requirements.txt](requirements.txt):

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
jupyter notebook
```

The notebooks locate the repository root by searching for `README.md`, `notebooks`, and `scripts`, so they can run from either:

- a normal `git clone`
- an extracted GitHub ZIP archive

## Interpretation Notes

- The synthetic regime hedge is an oracle benchmark: it knows the current simulated state at each hedge date.
- The cleaned synthetic write-up emphasizes dispersion and 5% tails more than 1% tails because the deepest tail estimates are noisier at moderate Monte Carlo sizes.
- The latent-state notebook is the practical follow-up to the oracle benchmark: it asks how much of the oracle advantage survives when the regime is hidden rather than observed.
- The final repo intentionally drops the earlier exploratory branches so the tracked content matches the final conclusions.
